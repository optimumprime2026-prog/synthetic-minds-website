# Design System v1 — Synthetic Minds

## Purpose
This design system defines the first visual and UI rules for the Synthetic Minds website. It is intended to keep the product consistent, high-trust, and enterprise-focused while remaining modern and technically sharp.

## Design principles
- Clarity over decoration
- Trust over hype
- Structure over noise
- Precision over playfulness
- Conversion over visual clutter

## Typography
### Font family
- Primary: Inter
- Alternative: Geist

### Type scale
- Display XL: 64px / 1.0 / 700
- Display L: 48px / 1.05 / 700
- Heading XL: 40px / 1.1 / 700
- Heading L: 32px / 1.15 / 700
- Heading M: 24px / 1.2 / 600
- Heading S: 20px / 1.3 / 600
- Body L: 18px / 1.6 / 400
- Body M: 16px / 1.6 / 400
- Body S: 14px / 1.5 / 400
- Caption: 12px / 1.4 / 500

## Layout system
### Container widths
- Max content width: 1200px
- Text content width: 760px
- Narrow content width: 640px

### Spacing scale
- 4px
- 8px
- 12px
- 16px
- 24px
- 32px
- 48px
- 64px
- 96px
- 128px

### Section spacing
- Desktop: 96px top / 96px bottom
- Tablet: 72px top / 72px bottom
- Mobile: 48px top / 48px bottom

## Grid
- 12-column desktop grid
- 8-column tablet grid
- 4-column mobile grid
- Prefer strong alignment and consistent gutters

## Buttons
### Primary button
- Background: Accent Blue
- Text: White
- Radius: 12px
- Padding: 14px 20px
- Weight: 600
- Hover: darker blue

### Secondary button
- Background: transparent
- Border: 1px solid border color
- Text: primary text color
- Radius: 12px
- Padding: 14px 20px

### Button rules
- Avoid too many button variants
- Keep CTA hierarchy obvious
- Use one primary CTA per section

## Cards
### Standard card
- Background: Surface Dark or Surface Light
- Border: subtle 1px border
- Radius: 20px
- Padding: 24px
- Shadow: minimal or none

### Use cases
- pain cards
- industry cards
- founder cards
- solution blocks

## Navigation
- Clean top nav
- Max 5 primary links
- Sticky or semi-sticky on scroll
- CTA button in top-right

### Suggested nav items
- Solutions
- Industries
- About
- Contact
- CTA: Book a call

## Forms
- Large clear labels
- Strong input contrast
- 12px–14px radius
- Inline validation states
- Clear success and error feedback

### Form rules
- Minimize fields
- Reduce friction
- Validate all inputs
- Keep trust cues near form CTA

## Sections
### Hero section
- Large headline
- Tight subheadline
- 1 primary CTA + 1 secondary CTA
- Clean visual support

### Problem / solution sections
- Use 3–4 cards max per row
- Keep copy compact
- Prefer clear labels over long paragraphs

### Founder section
- Two-column card layout
- Headshot optional
- 3–4 short proof points each

## Icons
- Use clean line icons or simple filled icons
- Keep icon style consistent
- Avoid overly playful icon sets

## Motion
### Use motion for
- fade-ins
- subtle upward reveal
- card hover feedback
- smooth section transitions
- progress / flow indication

### Avoid
- bounce animations
- excessive parallax
- flashy loaders
- long hero animations

## Border radius
- Small: 10px
- Medium: 12px
- Large: 20px
- XL section card: 24px

## Shadows
- Keep subtle
- Prefer borders + contrast over heavy shadows

### Example shadow
- `0 10px 30px rgba(0,0,0,0.12)` used sparingly

## Component priority list
Build these first:
1. Navbar
2. Hero
3. CTA buttons
4. Section wrapper
5. Card component
6. Industry cards
7. Founder cards
8. Footer
9. Contact form

## Accessibility notes
- Maintain strong color contrast
- Use semantic headings
- Ensure visible focus states
- Keep buttons and inputs touch-friendly
- Do not rely on motion alone for meaning

## Design system summary
The Synthetic Minds design system should feel clean, enterprise-grade, modern, and technically confident. The goal is not to impress with visual excess, but to build trust through clarity, consistency, and strong information hierarchy.
